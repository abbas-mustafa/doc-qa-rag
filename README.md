# DocQA — Chat With Your Documents

A full-stack Retrieval-Augmented Generation (RAG) app: upload PDFs, DOCX, or text files, and ask natural-language questions about them. Answers are generated by Gemini and grounded in your documents, with inline citations back to the source file.

Built with a Node/Express + PostgreSQL (pgvector) backend and a Next.js + Tailwind + Framer Motion frontend.

## Features

- **Workspaces** — group documents and conversations into isolated projects
- **Document upload** — drag-and-drop PDF / DOCX / TXT, parsed and embedded in the background
- **Semantic search** — pgvector cosine-similarity retrieval scoped per workspace
- **Cited answers** — every answer links back to the specific document (and page, where available) it was drawn from
- **Conversation history** — persisted per workspace
- **Modern UI** — animated gradient background, glassmorphism chat bubbles, drag-and-drop upload, live processing status, responsive layout (desktop 3-column, mobile collapsible panels)

## Architecture

```
frontend/   Next.js 16 (App Router, TypeScript, Tailwind CSS, Framer Motion)
backend/    Express API (Node.js, ESM)
            ├─ documentParser.js   PDF/DOCX/TXT text extraction
            ├─ chunker.js          overlapping text chunking
            ├─ embeddings.js       Gemini embeddings (768-dim)
            ├─ retrieval.js        pgvector similarity search
            └─ llm.js              Gemini chat completion with citation prompting
```

**Data flow:**

1. Upload → parse → chunk → embed → store vectors in Postgres (background job, status polled by the frontend)
2. Question → embed → retrieve top-K similar chunks (cosine similarity, workspace-scoped, threshold-filtered) → prompt Gemini with cited context → store + return the answer

## Getting started

### Prerequisites

- Node.js 18+
- A PostgreSQL database with the `pgvector` extension (e.g. a free [Supabase](https://supabase.com) project)
- A [Gemini API key](https://aistudio.google.com/apikey) (free tier available)

### 1. Backend

```bash
cd backend
npm install
cp .env.example .env   # fill in DATABASE_URL and GEMINI_API_KEY
npm run migrate         # creates tables + enables pgvector
npm run dev              # starts the API on http://localhost:5000
```

### 2. Frontend

```bash
cd frontend
npm install
echo "NEXT_PUBLIC_API_URL=http://localhost:5000" > .env.local
npm run dev               # starts the app on http://localhost:3000
```

Open `http://localhost:3000`, create a workspace, upload a document, and start asking questions.

## Environment variables

**`backend/.env`**

| Variable | Description |
|---|---|
| `PORT` | API server port (default `5000`) |
| `DATABASE_URL` | Postgres connection string (use the pooler/IPv4 host if on Supabase) |
| `GEMINI_API_KEY` | Google Gemini API key |
| `EMBEDDING_MODEL` | Defaults to `gemini-embedding-001` |
| `CHAT_MODEL` | Defaults to `gemini-2.5-flash` |
| `MAX_FILE_SIZE_MB` | Upload size limit (default `10`) |

**`frontend/.env.local`**

| Variable | Description |
|---|---|
| `NEXT_PUBLIC_API_URL` | Base URL of the backend API |

## API reference

| Method | Route | Description |
|---|---|---|
| `POST` | `/api/workspaces` | Create a workspace |
| `GET` | `/api/workspaces` | List workspaces |
| `GET` | `/api/workspaces/:id` | Get a workspace |
| `POST` | `/api/documents/upload/:workspaceId` | Upload + process a document |
| `GET` | `/api/documents/workspace/:workspaceId` | List documents in a workspace |
| `DELETE` | `/api/documents/:id` | Delete a document |
| `POST` | `/api/chat/:workspaceId` | Ask a question, get a cited answer |
| `GET` | `/api/chat/:workspaceId/history` | Get conversation history |

## Deployment notes

- **Frontend** deploys to Vercel with zero config.
- **Backend** needs a persistent Node host (e.g. Render, Railway, Fly.io) — it does background processing and local file uploads, which don't fit a serverless/edge model.
