# DocQA — Chat With Your Documents

A full-stack Retrieval-Augmented Generation (RAG) app: upload PDFs, DOCX, or text files, and ask natural-language questions about them. Answers are generated by Gemini and grounded in your documents, with inline citations back to the source file — **including figures, charts, and scanned pages**.

Built with a **Python / FastAPI** backend and a **Next.js + Tailwind + Framer Motion** frontend, over PostgreSQL (pgvector) and Google Gemini.

## Features

- **Workspaces** — group documents and conversations into isolated projects
- **Rich document parsing** — each file type handled distinctly:
  - **PDF** (PyMuPDF): text with real page numbers, tables extracted as markdown, embedded figures described by Gemini vision, and OCR (via vision) for scanned/image-only pages
  - **DOCX** (python-docx): heading structure preserved, tables, embedded images
  - **TXT**: plain text
- **Dynamic, structure-aware chunking** — chunk size adapts to document shape (list-like vs prose), tables/figures stay intact, splits respect heading → paragraph → sentence boundaries, with overlap carried for continuity
- **Semantic search** — pgvector cosine-similarity retrieval scoped per workspace
- **Cited answers** — every answer links back to the specific document + page it came from; token-optimized prompts (capped context, deduped sources)
- **Large uploads** — up to 100 MB, streamed to disk (no memory blow-up)
- **Multi-user ready** — Supabase-Auth JWT verification; every query is user-scoped
- **Resilient** — automatic retry/backoff on Gemini rate limits
- **Modern UI** — animated gradient background, glassmorphism chat bubbles, drag-and-drop upload, auto-growing input, live processing status, responsive layout

## Architecture

```
frontend/     Next.js 16 (App Router, TypeScript, Tailwind CSS, Framer Motion)
backend-py/   FastAPI (Python 3.14, async)
  app/
    main.py            app + lifespan + CORS + error handling
    config.py          pydantic-settings config
    db.py              async psycopg3 pool + pgvector
    auth.py            Supabase JWT verification (user scoping)
    schema.sql         DB schema  ·  migrate.py  applies it
    routers/           workspaces · documents · chat  (user-scoped)
    services/
      parsing.py       per-file-type extraction (PyMuPDF / python-docx) + tables
      vision.py        Gemini vision: figure descriptions + page OCR
      chunking.py      dynamic, structure-aware chunking
      embeddings.py    Gemini embeddings (768-dim, batched, retried)
      retrieval.py     pgvector similarity search
      llm.py           token-optimized, citation-aware generation
```

**Data flow:**

1. Upload → parse (text + tables + figures + OCR) → dynamic chunk → embed → store vectors in Postgres (background task; frontend polls status)
2. Question → embed → retrieve top-K similar chunks (cosine, workspace-scoped, threshold-filtered) → prompt Gemini with capped, cited context → store + return the answer

## Getting started

### Prerequisites

- Python 3.11+ (developed on 3.14)
- Node.js 18+
- A PostgreSQL database with `pgvector` (e.g. a free [Supabase](https://supabase.com) project)
- A [Gemini API key](https://aistudio.google.com/apikey)

### 1. Backend

```bash
cd backend-py
python -m venv .venv
.venv\Scripts\activate            # Windows  (source .venv/bin/activate on macOS/Linux)
pip install -r requirements.txt
copy .env.example .env            # fill in DATABASE_URL and GEMINI_API_KEY
python -m app.migrate             # creates tables + enables pgvector
python run.py                     # starts the API on http://localhost:8000
```

Interactive API docs (Swagger) at `http://localhost:8000/docs`.

### 2. Frontend

```bash
cd frontend
npm install
echo NEXT_PUBLIC_API_URL=http://localhost:8000 > .env.local
npm run dev                       # http://localhost:3000
```

## Environment variables

**`backend-py/.env`**

| Variable | Description |
|---|---|
| `PORT` | API port (default `8000`) |
| `CORS_ORIGINS` | Comma-separated allowed origins (default `http://localhost:3000`) |
| `DATABASE_URL` | Postgres connection string (use the Supabase pooler/IPv4 host) |
| `GEMINI_API_KEY` | Google Gemini API key |
| `EMBEDDING_MODEL` | Default `gemini-embedding-001` |
| `CHAT_MODEL` / `VISION_MODEL` | Default `gemini-2.5-flash` |
| `MAX_FILE_SIZE_MB` | Upload limit (default `100`) |
| `AUTH_ENABLED` | `true` to enforce Supabase JWTs (default `false` for local dev) |
| `SUPABASE_JWT_SECRET` | Supabase project JWT secret (when auth is enabled) |

**`frontend/.env.local`**

| Variable | Description |
|---|---|
| `NEXT_PUBLIC_API_URL` | Base URL of the backend API |

## API reference

| Method | Route | Description |
|---|---|---|
| `POST` | `/api/workspaces` | Create a workspace |
| `GET` | `/api/workspaces` | List the user's workspaces |
| `GET` | `/api/workspaces/{id}` | Get a workspace |
| `DELETE` | `/api/workspaces/{id}` | Delete a workspace (cascades) |
| `POST` | `/api/documents/upload/{workspaceId}` | Upload + process a document |
| `GET` | `/api/documents/workspace/{workspaceId}` | List documents |
| `DELETE` | `/api/documents/{id}` | Delete a document |
| `POST` | `/api/chat/{workspaceId}` | Ask a question, get a cited answer |
| `GET` | `/api/chat/{workspaceId}/history` | Conversation history |

## Notes

- **Free-tier Gemini** has strict rate/quota limits; large or figure-heavy documents can exceed the free embedding rate. The backend retries with backoff, but a paid Gemini key is recommended for heavy use.
- **Deployment**: frontend → Vercel (zero-config); backend → a persistent Python host (Render / Railway / Fly.io), since it does background processing and file uploads that don't fit a serverless model.
